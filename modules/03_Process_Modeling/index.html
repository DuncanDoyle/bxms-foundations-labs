Process Design

Tips for Good Design

  • Model business processes, not technical details

  • Modularize process definitions

  • Use domain-specific nodes to abstract technical details

  • Define relationship between rules and processes

    • Only embed rules inside process if rule life cycle is same as process life cycle

  • Use asynchronous execution for task that can fail

  • Use asynchronous continuation for long-running tasks
