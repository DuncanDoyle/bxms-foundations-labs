RHPAM Integration Considerations

Despite many integration points/strategies, recommend attempting to solely leverage KIE Execution Server REST endpoints

  • Maintains loose coupling

  • 7.x provides vast majority of needed data

  • Extensible through custom KIE Server capabilities

  • Tweak with custom queries functionality

RHPAM Integration Considerations

Widgets depend on defined knowledge source​, which essentially corresponds to KIE Execution Server instance

  • Admin app defines base endpoint + service credential

  • KIE Instance has one or more installed capabilities

    • Examples: KIE  Server, BPM, BPM-UI, BRM, BRP, DMN, CaseMgmt

    • Can programmatically determine if they are present

  • Admin portion of widget requires one or more capabilities
